A Node.JS Server that Won't Melt

Our Goal:

During extreme traffic we should stay responsive for as many users as possible and block the remainder.

Breaking Bad:


We want a drop-in solution to instantly make an existing server fail with grace.

We want to use it like this:


var toobusy = require('toobusy');

app.use(function(req, res, next) {
  if (toobusy()) res.send(503, "I'm busy right now, sorry.");
  else next();
});

Event Loop Lag!

By measuring how long it takes a job to get through the event loop you can get an understanding of how busy your process is, regardless of system load.

Implementation

Poll event loop every 500ms.

Calculate dampened current lag.

Configured lag high water mark.


    bool block = false;
    if (s_currentLag > HIGH_WATER_MARK_MS) {
        // probabilistically block requests proportional to how
        // far behind we are.
        double pctToBlock = ((s_currentLag - HIGH_WATER_MARK_MS) /
                             (double) HIGH_WATER_MARK_MS) * 100.0;
        double r = (rand() / (double) RAND_MAX) * 100.0;
        if (r < pctToBlock) block = true;
    }
    return block ? True() : False();
